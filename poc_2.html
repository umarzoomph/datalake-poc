Critical clarification

The corrected data model

S3 is not a peer data source. It carries large JSON enrichment payloads that reference Elasticsearch documents via a unique foreign key — es_ref_id. This changes the entire join and reconciliation strategy.

entity_id PK
tenant_id
status
created_at
updated_at — watermark key
raw_json — full _source
JOIN KEY
S3 large JSON — enrichment sidecar
es_ref_id → entity_id FK
tenant_id
enrichment_type
model_version
enrichment_at — version key
enrichment_json 100KB–10MB
Key design principle: The join is always es_raw LEFT JOIN s3_enrichment ON entity_id = es_ref_id — never a MERGE or UNION. An S3 payload arriving before its ES document is an orphan that needs monitoring, not an error. Every platform handles this as a LEFT JOIN null.
Orphan enrichment risk: When an S3 enrichment arrives before the ES document has been indexed (race condition common in async AI pipelines), the enrichment row has no parent. Always run orphan coverage monitoring queries to detect this pattern. Databricks Delta MERGE is the only platform that handles orphan reconciliation automatically via WHEN NOT MATCHED.

Platform deep dives

Architecture-specific implementation

Full pipeline design for each of the three recommended platforms, incorporating the corrected ES + S3 enrichment data model.

Snowflake + Snowpipe

ES canonical records ingest via Lambda scroll → S3 staging → Snowpipe into raw_es. S3 enrichment payloads land in s3_enrichment keyed by es_ref_id. RabbitMQ events trigger Lambda via SQS bridge.

$200–600/mo 1–5s queries
1
RabbitMQ → SQS bridge → Lambda trigger. Domain events (record.updated, enrichment.complete) are bridged from RabbitMQ to SQS. Lambda consumes and routes: ES events trigger the ES extractor; enrichment.complete events trigger the S3 transformer.
2
Lambda ES extractor → S3 staging. Uses search_after pagination with an SSM watermark. Writes Parquet to s3://staging/es/ prefix. Row group size 50k, Snappy compressed.
3
Snowpipe auto-ingest. SQS notification on S3 file arrival triggers Snowpipe. Copies into raw_es table. ON_ERROR = CONTINUE prevents bad rows blocking the pipe.
4
S3 enrichment processor. Extracts es_ref_id from large JSON payload, validates it exists, writes to s3://staging/enrichment/. Orphan detection logs any missing keys immediately.
5
dbt MERGE model. Joins raw_es LEFT JOIN s3_enrichment ON entity_id = es_ref_id. Produces entities_unified — the single source of truth for all dashboards.

ClickHouse + ReplacingMergeTree

Fastest query performance of all six options. ES data lands in es_raw via S3Queue. Enrichment in s3_enrichment via second S3Queue. AggregatingMergeTree materialized view pre-joins for sub-second dashboard queries.

$80–250/mo <1s queries
1
ReplacingMergeTree engine. Both es_raw and s3_enrichment use ReplacingMergeTree with updated_at as the version column. ClickHouse keeps the highest-version row per ORDER BY key during background merges.
2
S3Queue engine. ClickHouse polls S3 staging prefixes continuously. Each file processed exactly once, tracked in ZooKeeper. Materialized view automatically pipes rows from queue into es_raw and s3_enrichment tables.
3
AggregatingMergeTree serving layer. entities_serving is pre-joined using two MVs — one fired on each ES insert, one on each enrichment insert. Dashboard queries hit this table with no FINAL and return in milliseconds.
4
FINAL modifier for correctness. Until background merges run, duplicate rows coexist in raw tables. For exact counts, append FINAL to raw table queries or use the serving layer MV which is already deduplicated.
5
Orphan monitoring. LEFT JOIN query on s3_enrichment LEFT JOIN es_raw WHERE es_raw.entity_id IS NULL surfaces all enrichments without a matching ES document. Schedule this as a daily alert.

ReplacingMergeTree is eventually consistent. This is a feature, not a bug — inserts are always fast because deduplication happens asynchronously. For reporting workloads where occasional duplicates during a merge window are acceptable, this is ideal. For billing or exact-count use cases, always use FINAL or the serving layer MV.

Databricks + Delta Lake

Best choice when ML pipelines are in the roadmap. Auto Loader continuously ingests ES Parquet from S3. Delta MERGE handles deduplication and orphan enrichment reconciliation. SQL Warehouse delivers 2–6s dashboards.

$300–900/mo 2–6s queries
1
Auto Loader for ES data. Databricks Auto Loader monitors the S3 staging prefix using cloudFiles source. Processes only new files using an incremental checkpoint. Schema inference handles ES field additions automatically.
2
Delta MERGE for deduplication. MERGE INTO es_raw WHEN MATCHED AND source.updated_at > target.updated_at THEN UPDATE. Immediate ACID guarantee — no eventual consistency window like ClickHouse.
3
Orphan enrichment handling. The Delta MERGE model for enrichments uses WHEN NOT MATCHED BY TARGET THEN INSERT to hold orphan enrichments. A scheduled reconciliation job re-attempts joining them against es_raw after a configurable delay.
4
Unified entity view. entities_unified is a Delta table maintained by a Databricks workflow joining ES and enrichment tables on entity_id = es_ref_id. Unity Catalog governs access and lineage.
5
Databricks SQL Serverless. Dashboards connect via JDBC to a Serverless SQL Warehouse. No cluster warm-up time. Results cache handles repeated dashboard queries. Cost charged per DBU consumed, not per hour.

Implementation code

Key code across all platforms

Production-ready code for the most critical components: ES extractor, S3 enrichment processor, and deduplication patterns.

Lambda — ES incremental extractor (shared across all platforms)
Python 
# Shared across Snowflake, ClickHouse, Databricks, and Athena stacks
import os, json, boto3, pyarrow as pa, pyarrow.parquet as pq
from datetime import datetime, timezone
from io import BytesIO
from elasticsearch import Elasticsearch

ES_HOST        = os.environ["ES_HOST"]
STAGING_BUCKET = os.environ["STAGING_BUCKET"]
WATERMARK_KEY  = "/datalake/watermark/es"
PAGE_SIZE      = 2000

es  = Elasticsearch(ES_HOST)
ssm = boto3.client("ssm")
s3  = boto3.client("s3")

def get_watermark() -> str:
    try:
        return ssm.get_parameter(Name=WATERMARK_KEY)["Parameter"]["Value"]
    except ssm.exceptions.ParameterNotFound:
        return "1970-01-01T00:00:00Z"

def scroll_es(since: str) -> list:
    hits, search_after = [], None
    query = {
        "size": PAGE_SIZE,
        "query": {"range": {"updated_at": {"gt": since}}},
        "sort": [{"updated_at": "asc"}, {"_id": "asc"}]
    }
    while True:
        if search_after:
            query["search_after"] = search_after
        resp = es.search(index="*", body=query)
        page = resp["hits"]["hits"]
        if not page: break
        hits.extend(page)
        search_after = page[-1]["sort"]
        if len(page) < PAGE_SIZE: break
    return hits

def handler(event, context):
    trigger = event.get("trigger_source", "schedule")
    since   = event.get("updated_after") or get_watermark()
    run_id  = datetime.now(timezone.utc).strftime("%Y%m%dT%H%M%SZ")
    hits    = scroll_es(since)
    if not hits:
        return {"status": "no_new_records"}

    rows = [{
        "entity_id":    h["_id"],
        "es_index":     h["_index"],
        "updated_at":   h["_source"].get("updated_at", since),
        "tenant_id":    h["_source"].get("tenant_id", ""),
        "status":       h["_source"].get("status", ""),
        "pipeline_run": run_id,
        "raw_json":     json.dumps(h["_source"]),
    } for h in hits]

    buf = BytesIO()
    pq.write_table(pa.Table.from_pylist(rows), buf,
                   compression="snappy", row_group_size=50_000)
    key = f"es/{run_id}/batch_{len(hits)}.parquet"
    s3.put_object(Bucket=STAGING_BUCKET, Key=key, Body=buf.getvalue())

    if trigger == "schedule":
        latest = max(h["_source"].get("updated_at", since) for h in hits)
        ssm.put_parameter(Name=WATERMARK_KEY, Value=latest,
                          Type="String", Overwrite=True)
    return {"status": "ok", "records": len(hits)}
Lambda — S3 enrichment processor (extracts es_ref_id foreign key)
Python 
# Handles large JSON enrichment files — extracts es_ref_id for the join
import os, json, boto3, pyarrow as pa, pyarrow.parquet as pq
from io import BytesIO
from datetime import datetime, timezone

STAGING_BUCKET = os.environ["STAGING_BUCKET"]
ES_REF_FIELD   = os.environ.get("ES_REF_FIELD", "es_ref_id")
s3 = boto3.client("s3")

def parse_enrichment(doc: dict, s3_key: str) -> dict:
    # Try multiple common field name patterns for the reference key
    es_ref_id = (
        doc.get(ES_REF_FIELD)
        or doc.get("entity_id")
        or doc.get("doc_id")
        or doc.get("record_id")
    )
    if not es_ref_id:
        raise ValueError(
            f"Missing ES reference key in {s3_key}. "
            f"Keys found: {list(doc.keys())[:10]}"
        )
    return {
        "es_ref_id":          str(es_ref_id),   # FK → es_raw.entity_id
        "tenant_id":          str(doc.get("tenant_id", "")),
        "enrichment_version": str(
            doc.get("enrichment_at") or
            doc.get("updated_at") or
            datetime.now(timezone.utc).isoformat()
        ),
        "source_s3_key":      s3_key,
        "model_version":      str(doc.get("model_version", "")),
        "enrichment_type":    str(doc.get("type", "ai_enrichment")),
        "enrichment_json":    json.dumps(doc),   # full payload preserved
    }

def handler(event, context):
    sources = []
    if "Records" in event:  # S3 event notification
        sources = [{"bucket": r["s3"]["bucket"]["name"],
                    "key": r["s3"]["object"]["key"]} for r in event["Records"]]
    elif event.get("trigger_source") == "rabbitmq":  # enrichment.complete event
        sources = [{"bucket": os.environ["AI_SOURCE_BUCKET"],
                    "key": event["s3_key"]}]

    rows, errors = [], []
    run_id = datetime.now(timezone.utc).strftime("%Y%m%dT%H%M%SZ")

    for src in sources:
        raw = json.loads(s3.get_object(**src)["Body"].read())
        docs = raw if isinstance(raw, list) else [raw]
        for doc in docs:
            try:
                rows.append(parse_enrichment(doc, src["key"]))
            except ValueError as e:
                errors.append(str(e)); print(f"ORPHAN: {e}")

    if rows:
        buf = BytesIO()
        pq.write_table(pa.Table.from_pylist(rows), buf,
                       compression="snappy", row_group_size=10_000)
        key = f"s3_enrichment/{run_id}/batch_{len(rows)}.parquet"
        s3.put_object(Bucket=STAGING_BUCKET, Key=key, Body=buf.getvalue())
    return {"status": "ok", "rows": len(rows), "errors": len(errors)}
Snowflake — table DDL, Snowpipe, and unified entity view
SQL 
-- ES canonical table
CREATE TABLE datalake.raw_es (
    entity_id     STRING     NOT NULL,
    tenant_id     STRING,
    updated_at    TIMESTAMP_NTZ,
    ingested_at   TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP(),
    status        STRING,
    raw_json      VARIANT     -- flexible, handles schema evolution
);

-- S3 enrichment sidecar table (keyed by es_ref_id FK)
CREATE TABLE datalake.s3_enrichment (
    es_ref_id         STRING     -- FK → raw_es.entity_id
    tenant_id         STRING,
    enrichment_version TIMESTAMP_NTZ,
    source_s3_key     STRING,
    model_version     STRING,
    enrichment_type   STRING,
    enrichment_json   VARIANT   -- large JSON stored as VARIANT
);

-- Snowpipe: auto-ingest ES Parquet from S3 staging
CREATE PIPE pipe_es AUTO_INGEST = TRUE AS
    COPY INTO raw_es (entity_id, tenant_id, updated_at, status, raw_json)
    FROM (
        SELECT $1:entity_id::STRING, $1:tenant_id::STRING,
               $1:updated_at::TIMESTAMP_NTZ, $1:status::STRING, $1
        FROM @raw_stage_es
    )
    FILE_FORMAT = (TYPE = PARQUET) ON_ERROR = 'CONTINUE';

-- Unified entity view — LEFT JOIN on the reference key
CREATE VIEW datalake.entities_unified AS
SELECT
    e.entity_id,
    e.tenant_id,
    e.status,
    e.updated_at                          AS es_updated_at,
    e.raw_json                            AS es_payload,
    s.enrichment_json                     AS s3_payload,   -- NULL if no enrichment yet
    s.model_version,
    s.enrichment_version                  AS s3_updated_at,
    s.source_s3_key,
    (s.es_ref_id IS NOT NULL)            AS has_enrichment
FROM datalake.raw_es e
LEFT JOIN datalake.s3_enrichment s
    ON  e.entity_id = s.es_ref_id
    AND e.tenant_id = s.tenant_id;

-- Orphan coverage monitoring query
SELECT tenant_id,
    COUNT(DISTINCT entity_id)                      AS total_entities,
    COUNT(DISTINCT CASE WHEN has_enrichment THEN entity_id END) AS enriched,
    ROUND(enriched / total_entities * 100, 1)        AS coverage_pct
FROM datalake.entities_unified
GROUP BY tenant_id;
ClickHouse — ReplacingMergeTree DDL and serving layer
SQL 
-- ES raw table: ReplacingMergeTree keeps highest updated_at per entity_id
CREATE TABLE datalake.es_raw (
    entity_id    String,
    tenant_id    String,
    updated_at   DateTime64(3),   -- ReplacingMergeTree version key
    ingested_at  DateTime64(3) DEFAULT now64(),
    status       String,
    raw_json     String
) ENGINE = ReplacingMergeTree(updated_at)
  PARTITION BY toYYYYMM(ingested_at)
  ORDER BY (tenant_id, entity_id);

-- S3 enrichment table: keyed by es_ref_id (FK to es_raw.entity_id)
CREATE TABLE datalake.s3_enrichment (
    es_ref_id         String,   -- FK → es_raw.entity_id
    tenant_id         String,
    enrichment_version DateTime64(3),
    source_s3_key     String,
    model_version     String,
    enrichment_type   String,
    enrichment_json   String
) ENGINE = ReplacingMergeTree(enrichment_version)
  PARTITION BY toYYYYMM(ingested_at)
  ORDER BY (tenant_id, es_ref_id);  -- same structure as es_raw

-- Serving layer: pre-joined AggregatingMergeTree for sub-second queries
CREATE TABLE datalake.entities_serving (
    entity_id       String,
    tenant_id       String,
    status          SimpleAggregateFunction(anyLast, String),
    es_updated_at   SimpleAggregateFunction(max, DateTime64(3)),
    s3_updated_at   SimpleAggregateFunction(max, DateTime64(3)),
    es_payload      SimpleAggregateFunction(anyLast, String),
    s3_payload      SimpleAggregateFunction(anyLast, String),  -- NULL until enrichment arrives
    model_version   SimpleAggregateFunction(anyLast, String)
) ENGINE = AggregatingMergeTree()
  ORDER BY (tenant_id, entity_id);

-- Dashboard query: no FINAL needed, sub-second
SELECT
    entity_id,
    anyLastMerge(status)        AS status,
    maxMerge(es_updated_at)     AS es_updated_at,
    notEmpty(anyLastMerge(s3_payload)) AS has_enrichment
FROM datalake.entities_serving
WHERE tenant_id = 'acme'
GROUP BY entity_id, tenant_id
ORDER BY maxMerge(es_updated_at) DESC
LIMIT 1000;

Decision guide

Which platform to choose

A simple decision tree based on your actual requirements — not preferences.

Step 1 — Do dashboards need data fresher than 5 minutes?
If yes → evaluate Arch 4 (Kafka + Iceberg). If no → continue.
Step 2 — Is ML / feature engineering on the roadmap in the next 12 months?
If yes → Databricks (Delta Lake + MLflow unified platform). If no → continue.
Step 3 — Do you need row-level security, data sharing, or enterprise audit logs?
If yes → Snowflake (governance champion). If no → continue.
Step 4 — Do dashboards need sub-second response for 50+ concurrent users?
If yes → ClickHouse (fastest query engine in this evaluation). If no → continue.
Default → S3 + Athena (Arch 1).
Meets all stated requirements (<30s dashboards, millions of records/month, incremental loading, schema evolution, enrichment join via es_ref_id) at $10–40/month. Start here. Migrate when a specific KPI proves the need.
